Hardcoded Secrets / API Keyleri

KRİTİK
Google Maps API Key – Açık Kaynak Kodunda
Google Maps API keyi doğrudan JavaScript bundle'ına bölünmemiş şekilde gömülü. Bu key; geocoding ve nearbysearch API çağrılarında birden fazla yerde tekrar edilmektedir. Google, Play Store'da tespit ettiğinde bu keyi otomatik olarak revoke edebilir veya uygulamayı reddedebilir.
VITE_GOOGLE_MAPS_API_KEY: "AIzaSyCGRGdSA0IZHxgGI4PCv00kQ8xJ5dpx7Gc"
🔧 Düzeltme: API keyini Google Cloud Console'da kısıtlayın (Android app referrer + SHA-1 ile). Uzun vadeli çözüm olarak key'i bir backend proxy üzerinden kullanın.
Supabase Anon Key – Açık Kaynak Kodunda
Supabase anonim anahtarı ve tam Supabase URL'si JS bundle'ına gömülü. Bu anahtarla herhangi biri doğrudan veritabanına sorgu yapabilir; eğer Row Level Security (RLS) yanlış yapılandırılmışsa bu ciddi bir veri güvenlik açığı oluşturabilir.
VITE_SUPABASE_URL: "https://xmskjcdwmwlcmjexnnxw.supabase.co" VITE_SUPABASE_ANON_KEY: "sb_publishable_aJ6U77lzPTrGZEjQ6WJgUw_PPusa7R-"
🔧 Düzeltme: Supabase anon key teknik olarak public-facing tasarlanmış olmakla birlikte, RLS kurallarınızın doğru ve sıkı olduğundan emin olunuz. Hassas verileri mutlaka RLS ile koruyun.
Local Dev URL – Production Bundle'da
Geliştirme ortamına ait bir yerel ağ IP adresi production build'ine gömülü kalmış. Bu hem bir güvenlik açığı hem de kullanıcı deneyimi sorunudur – production'da bu URL'ye ulaşılamaz.
VITE_API_URL: "http://192.168.3.13:5000/api"
🔧 Düzeltme: .env dosyalarında ortama göre farklı URL kullanın. Production build için .env.production dosyasında gerçek API URL'sini belirleyin ve vite build sırasında doğru env dosyasının yüklendiğini doğrulayın.

Network Security & Cleartext Traffic

UYARI
usesCleartextTraffic = true (Manifest)
AndroidManifest.xml'de usesCleartextTraffic="true" ayarı var. Bu, uygulamanın HTTP (şifresizilenmiş) trafiğe izin verdiğini gösterir. Android 9+ (API 28+) varsayılan olarak bu trafiği engeller, ancak bu flag o engeli kaldırır.
🔧 Düzeltme: Tüm API endpoint'lerini HTTPS'e taşıdıktan sonra bu flag'ı kaldırın veya false yapın.
network_security_config.xml'de Geniş Kapsamlı Cleartext İzinleri
network_security_config dosyasında localhost, 127.0.0.1, 10.0.2.2 ve birçok 192.168.x.x alt ağı için cleartext (HTTP) trafiğe izin verilmiş. Bazıları (emulator: 10.0.2.2, localhost) geliştirme amaçlı olabilir ancak 192.168.x.x alt ağlarının production'da kalması risk taşır.
🔧 Düzeltme: Sadece gerekli geliştirme adresleri için (localhost, 10.0.2.2) cleartext izni verin. 192.168.x.x satırlarını kaldırın.

Kamera İzni Manifest'te Yok – Ancak Capacitor Camera Plugin Kullanılıyor
Capacitor Camera plugin aktif olmasına rağmen android.permission.CAMERA doğrudan Manifest'te listelenmemiş. Capacitor runtime'da dinamik olarak talep edebilir, ancak bazı cihazlarda veya Android versiyon öncelerinde bu yetersiz kalabilir. Ayrıca Play Store'da bu iznin açıkça belirtilmesi beklenir.
🔧 Düzeltme: AndroidManifest.xml'e <uses-permission android:name="android.permission.CAMERA"/> ekleyin.
Location Permissions – Capacitor Config'de Doğru Açıklanmış
Geolocation plugin'in permission mesajları kullanıcıya neden konum bilgisinin gerekli olduğunu Türkçe olarak açıkça anlatmaktadır. Bu Play Store'un "Açıklama Politikası" için olumlu bir nokta.

Exported Components

UYARI
MainActivity – exported="true" (Normal – Launcher Activity)
MainActivity'nin exported="true" olması normal çünkü LAUNCHER intent-filter var. Ancak aynı activity'de scheme="com.esnaftaucuz.app" ile bir custom deep link intent-filter daha var. Bu, dışarıdan bu activity'ye intent gönderilmesine olanak tanır. Deep link data doğrulaması olmalı.
🔧 Düzeltme: Deep link intent-filter'ın geldiği verileri validate edin. Gerekirse ayrı bir Activity kullanın ve exported="false" yapın.
ProfileInstallReceiver – exported="true" (androidx)
androidx.profileinstaller.ProfileInstallReceiver exported=true ile gelmiş ancak android.permission.DUMP ile korunmuştur. Bu bir androidx kütüphane standartı olduğundan normal kabul edilebilir, ancak production'da profil installer aktivitelerinin gerekliliği sorgulanabilir.
FileProvider & InitializationProvider – exported="false" ✓
Her iki provider da doğru şekilde exported="false" ile tanımlanmış.

Debug Artefaktlar

UYARI
DebugProbesKt.bin – Kotlin Coroutine Debug Dosyası
DebugProbesKt.bin dosyası release build'de mevcut. Bu Kotlin Coroutine debug probes ile ilgilidir. Release build'de genellikle bu dosya olmayacaktır; varlığı, build'in debug modunda yapıldığını ya da probes kaldırma optimizasyonunun eksik olduğunu düşündürür.
🔧 Düzeltme: build.gradle'da kotlinOptions { freeCompilerArgs += "-Xdebug" } satırının olmadığını doğrulayın. Kotlin coroutine debug probes'u production'da devre dışı bırakın.
