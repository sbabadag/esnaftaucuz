# Google OAuth Consent Screen Customization Guide

This guide explains how to customize the Google OAuth consent screen that appears when users click "Sign in with Google".

## 📋 Overview

The Google OAuth consent screen is the page that Google shows to users when they authorize your app. This is configured in **Google Cloud Console**, not in your code.

## 🔧 Configuration Steps

### 1. Access Google Cloud Console

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Select your project (or create a new one)
3. Navigate to **APIs & Services** → **OAuth consent screen**

### 2. Configure OAuth Consent Screen

#### App Information
- **App name**: `esnaftaucuz` (or your app name)
- **User support email**: Your support email
- **App logo**: Upload your app logo (optional, 120x120px recommended)
- **Application home page**: Your website URL
- **Application privacy policy link**: Link to your privacy policy
- **Application terms of service link**: Link to your terms of service
- **Authorized domains**: Add your domain (e.g., `selahattinbabadag.com`)

#### Developer Contact Information
- **Email addresses**: Your developer email

### 3. Configure Scopes

The scopes determine what information your app requests from Google:

**Current scopes in code:**
- `email` - User's email address
- `profile` - User's basic profile information

**To add custom scopes:**
1. In OAuth consent screen, go to **Scopes** section
2. Click **Add or Remove Scopes**
3. Select the scopes you need:
   - `email` - User's email address
   - `profile` - User's basic profile (name, picture)
   - `openid` - OpenID Connect (usually included automatically)

### 4. Configure OAuth Client in Supabase

1. Go to [Supabase Dashboard](https://app.supabase.com/)
2. Select your project
3. Navigate to **Authentication** → **Providers**
4. Click on **Google**
5. Configure:
   - **Client ID**: From Google Cloud Console
   - **Client Secret**: From Google Cloud Console
   - **Redirect URL**: Supabase will provide this (e.g., `https://your-project.supabase.co/auth/v1/callback`)

### 5. Add Redirect URI in Google Cloud Console

1. Go to **APIs & Services** → **Credentials**
2. Click on your OAuth 2.0 Client ID
3. Under **Authorized redirect URIs**, add:
   - `https://your-project.supabase.co/auth/v1/callback`
   - For mobile: `com.esnaftaucuz.app://` (if using custom URL scheme)

## 🎨 Customizing the Consent Screen Text

### App Name
- **Location**: OAuth consent screen → App information
- **What users see**: "esnaftaucuz wants to access your Google Account"
- **Customize**: Change the app name in Google Cloud Console

### Privacy Policy & Terms
- **Location**: OAuth consent screen → App information
- **What users see**: Links at the bottom of the consent screen
- **Customize**: Add your privacy policy and terms of service URLs

### Scopes Description
- **Location**: OAuth consent screen → Scopes
- **What users see**: "See your email address" and "See your personal info"
- **Customize**: These are automatically generated by Google based on scopes

## 🔐 Code Configuration

### Current OAuth Options (in `app/services/supabase-api.ts`)

```typescript
const oauthOptions: any = {
  queryParams: {
    access_type: 'offline',  // Request refresh token
    prompt: 'consent',       // Always show consent screen
  },
};
```

### Available Query Parameters

You can customize the OAuth flow by modifying `queryParams`:

- `prompt`: 
  - `'consent'` - Always show consent screen (current)
  - `'select_account'` - Show account selector
  - `'none'` - Don't show UI if user is already logged in
  - `'consent select_account'` - Show both

- `access_type`:
  - `'offline'` - Request refresh token (current)
  - `'online'` - Only request access token

- `hd`: Restrict to specific Google Workspace domain
  - Example: `hd: 'company.com'`

- `login_hint`: Pre-fill email
  - Example: `login_hint: 'user@example.com'`

### Example: Customized OAuth Options

```typescript
const oauthOptions: any = {
  queryParams: {
    access_type: 'offline',
    prompt: 'consent select_account',  // Show consent and account selector
    hd: 'yourdomain.com',              // Restrict to specific domain (optional)
  },
  scopes: 'email profile',              // Explicitly set scopes (optional)
};
```

## 📱 Mobile Configuration

For mobile apps, you may also need to configure:

1. **Custom URL Scheme**: Already configured as `com.esnaftaucuz.app://`
2. **iOS URL Scheme**: Add to `Info.plist`
3. **Android Intent Filter**: Add to `AndroidManifest.xml`

   - Your app name
   - Your logo (if uploaded)
   - Correct scopes
   - Privacy policy and terms links

## 🚨 Common Issues

### "redirect_uri_mismatch" Error
- **Cause**: Redirect URI not added in Google Cloud Console
- **Fix**: Add Supabase callback URL to authorized redirect URIs

### Consent Screen Not Showing Custom Logo
- **Cause**: Logo not approved by Google
- **Fix**: Wait for Google review (can take 24-48 hours)

### Wrong App Name Showing
- **Cause**: App name not updated in Google Cloud Console
- **Fix**: Update app name in OAuth consent screen settings

After making changes in Google Cloud Console:
1. Changes may take a few minutes to propagate
2. Users may need to re-authorize if scopes changed
3. Test in incognito/private browsing mode to see changes immediately
